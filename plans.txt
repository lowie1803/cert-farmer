# Scale CCNA Course to Full 200-301 v1.1 Syllabus

## Context

CertFarmer currently has 5 modules covering a fraction of the CCNA 200-301 syllabus. The goal is to expand to the full 6-domain syllabus (~20 modules), add midterm/final exams, and build a quiz bank of ~230 questions. No usable free JSON quiz bank exists — we'll generate our own via a Claude API script.

**Reference resources for content accuracy:**
- [Jeremy's IT Lab](https://courses.jeremysitlab.com/p/ccna) — free 63-day YouTube course, community notes at github.com/psaumur/CCNA_Course_Notes
- [NetworkLessons.com](https://networklessons.com/cisco/ccna-200-301) — 100+ lessons in 13 units
- [Study-CCNA.com](https://study-ccna.com/) — 100+ free tutorials, 13 chapters

## Step 1: Add exam infrastructure (code changes)

### 1a. ExamView component — `src/components/ExamView.jsx` (new, ~200 lines)

Differences from QuizView:

| Feature | QuizView | ExamView |
|---------|----------|----------|
| Feedback | After each question | Only at end |
| Questions | 4-5 | 40-60 |
| Timer | None | Countdown (45/90 min) |
| Navigation | Sequential | Question palette (jump, flag for review) |
| Pass threshold | 70% | 80% (matches Cisco) |
| Order | Fixed | Shuffled per attempt |

**Phases:** intro → active → review → results
- `intro`: exam info + "Start Exam" button
- `active`: question display + navigation grid + timer + flag button
- `review`: summary of answered/unanswered/flagged, confirm submit
- `results`: score, per-domain breakdown (optional `domain` field on questions), pass/fail, review explanations

## Step 3: Content generation tooling

### 3a. `scripts/syllabus-outline.json` — topic definitions

Manually authored file defining what each module/lesson should cover:
```json
{
  "modules": {
    "cabling-and-topology": {
      "domain": 1,
      "icon": "🔌",
      "description": "Physical infrastructure and cabling",
      "lessons": {
        "cabling-types": {
          "title": "Cabling Types & Topology",
          "subtopics": ["UTP Cat5e/6/6a", "straight-through vs crossover", "single-mode vs multi-mode fiber", "PoE 802.3af/at/bt"]
        },
        "wireless-principles": {
          "title": "Wireless Principles",
          "subtopics": ["802.11 standards", "2.4 vs 5 GHz", "RF channels", "SSID/BSS/ESS"]
        }
      },
      "quizCount": 5
    }
  },
  "exams": {
    "midterm-1": { "domains": [1,2], "questionCount": 40, "timeLimit": 45 },
    "midterm-2": { "domains": [3,4], "questionCount": 40, "timeLimit": 45 },
    "final-exam": { "domains": [1,2,3,4,5,6], "questionCount": 60, "timeLimit": 90 }
  }
}
```

### 3b. `scripts/generate-content.js` — Claude API content generator

Node script (requires `ANTHROPIC_API_KEY` env var) that:
1. Reads `syllabus-outline.json`
2. For each lesson without a `.md` file → calls Claude API with a structured prompt → writes `.md`
3. For each module without a quiz `.json` → generates quiz questions → writes `.json`
4. For each exam → generates questions with `domain` tags → writes exam `.json`
5. Creates `module.json` for any new module directories
6. Idempotent — skips files that already exist (run incrementally)

**Dependency:** `npm install @anthropic-ai/sdk` (dev dependency)

### 3c. `scripts/validate-content.js` — content validator

Checks all content files for:
- Every lesson in module.json has a matching .md or .json file
- Quiz schema validity (4 options, correct in 0-3, explanation present)
- No duplicate question IDs within a file
- Exam questions have `domain` field
- Minimum word count on .md files (200+ words)
